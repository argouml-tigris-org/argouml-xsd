Development Phase

Mission:

  • Implement a UML profile for W3C simple XSD data types.
  • Write Java code to generate XSD files from UML Class Diagrams.
  • Write Java code to reverse engineer XSD files into UML Class Diagrams.

Related Resources:

  • Register issues in the ArgoUML Izzuezilla
  • Work with issues in the ArgoUML Issuezilla.
  • Please see the ArgoUML Wiki pages for the currently proposed design.

Development Examples:

The design has been initially deployed using XSLT to convert the Argo-provided XMI into an XSD based on the rules outlined in the design. Initial implementation REQUIRES the use of the xsd.xmi profile which must be added to the project and used EXCLUSIVELY for all primitive data types. The following are the related files:

  • xmi.xsd - REQUIRED to be added as a profile.
  • XmiToXsd.xsl - XLST used to convert XMI exported by Argo into and XSD file. Currently assumes the XSLT is in the same physical folder as the xmi.xsd file.
  • CourseExampleModel.zargo - An example data model rendered in ArgoUML to prove out the different scenarios outlined in the design document on the Wiki.
  • CourseExampleModel.xmi - Exported XMI from the .zargo file above.
  • CourseExampleModel.xsd - Generated output from applying the XSLT to the XMI file above.
